# 🧠 Архитектура: AI Life Assistant

Это приложение представляет собой умного персонального ассистента для трекинга жизни (финансов, задач, спорта и заметок) через естественный язык. Вместо того чтобы вручную заполнять десятки форм, пользователь просто "общается" с приложением (голосом или текстом), а AI-ядро само понимает намерение, классифицирует данные, извлекает цифры/даты и бережно раскладывает всё по базе.

## 🛠 Технологический стек

### Frontend (Клиент)
- **Фреймворк:** React (Vite) + TypeScript  
- **Роутинг:** React Router DOM
- **Стилизация:** Инлайн-стили + Tailwind classes 
- **Графики:** Recharts
- **Ключевые фичи:** 
  - Web Speech API для голосового ввода.
  - Кастомные бесконечные календари (Infinite scroll).
  - Оптимистичные UI-обновления (мгновенное переключение статусов).

### Backend (Сервер)
- **Платформа:** Node.js + Express + TypeScript
- **База Данных:** PostgreSQL (развернута в Supabase)
- **ORM:** Prisma
- **AI Интеграция:** Groq SDK (LLM: `llama-3.3-70b-versatile` — обеспечивает очень быстрый парсинг).
- **Аутентификация:** JWT токен.

## 🔑 Ключевые возможности (Features)

1. **Естественный ввод логов (AI Parsing)** 
   - Вы пишете: *"завтра купить молоко на 100 рублей и пробежать 5 км"*.
   - Нейросеть преобразует текст в три записи: 
     - Задачу (дедлайн на завтра), 
     - Финансы (расход 100 рублей), 
     - Спорт (бег 5 км).
   - Под капотом AI модель имеет строгий system prompt, заставляющий её возвращать JSON-объекты со структурированными данными.

2. **Q&A по своим данным**
   - Роутинг вопросов: Если пользователь что-то спрашивает (*"Сколько я потратил в этом месяце?"*), AI определяет (classify), что это вопрос "scope: finance".
   - Сервер вытягивает записи нужной категории из БД (PostgreSQL -> Prisma), формирует компактный JSON и скармливает его LLM для генерации человекочитаемого ответа.

3. **Аналитика (Dashboard)**
   - Прогресс-бар расхода бюджета за месяц в реальном времени.
   - Круговая диаграмма активностей по категориям.
   - Графики трат и спортивных походов по дням.
   - Учет отложенных/ждущих выполнения задач (`pending tasks`).

4. **Дневник / Управление записями (Records)**
   - Режим календаря и классического списка. 
   - Поиск по всем логам (клиентская фильтрация).
   - Экспорт всей истории в CSV-файл для Excel.
   - Закрытие задач или перенос дедлайнов в 1 клик.

5. **AI-Дайджесты**
   - Возможность получить консолидированную сводку за последние 7 дней от нейросети в виде одного сообщения.

## 📂 Структура базы данных (Prisma)

Архитектура хранения данных использует паттерн **Базовая запись + Расширения** для гибкости:

- **`User`** — пользователи системы.
- **`Record`** — ядро любой записи. Хранит общие данные: `userId`, `category`, `summary`, `details`, `emoji`, `createdAt`.

К `Record` опционально (один-к-одному) могут привязываться таблицы для конкретных типов данных:
- **`Finance`**: хранит сумму, валюту, тип (доход/расход).
- **`Sport`**: хранит дистанцию, подходы, повторения, длительность.
- **`Task`**: хранит дедлайн, приоритет, статус (выполнено или нет).
- **`Note`**: хранит специфичные для заметок поля (например, закрепление).

Также есть независимая модель:
- **`Budget`** — лимиты трат по категориям для каждого пользователя на месяц.

С такой архитектурой новые типы трекинга (например, "сон" или "настроение") добавляются созданием одной новой таблицы-расширения, не меняя и не засоряя основное ядро `Record`.

## 🔄 Жизненный цикл типичного запроса

1. Пользователь надиктовывает текст ➡️ Web Speech API преобразует голос в текст ➡️ Текст отправляется на `/api/parse`.
2. Контроллер обращается к Groq (llama-3.3-70b-versatile) ➡️ получает JSON массив записей.
3. Сервер отдает JSON клиенту, а клиент рендерит карточки. Пользователь их просматривает и жмет «Сохранить».
4. Клиент стучится в `/api/records` (POST). Сервер валидирует JWT-токен и пишет данные транзакцией через Prisma в БД.
5. Инвалидируется локальный кеш в React-приложении, и UI обновляется.
